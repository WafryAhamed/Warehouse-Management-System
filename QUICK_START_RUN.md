## 🐛 TROUBLESHOOTING

### Backend won't start
```powershell
# 1. Check if PostgreSQL is running
Get-Service postgresql-x64-*

# 2. Check if port 8080 is in use
netstat -ano | findstr ":8080"

# 3. Kill existing process on 8080
netstat -ano | findstr ":8080"
taskkill /PID <PID> /F

# 4. Rebuild and start again
cd "E:\Inventory Management System\InventoryServer"
mvn clean package -DskipTests
java -jar target\inventory-backend-1.0.0.jar
```

### Frontend won't connect
```powershell
# 1. Verify backend is running
Invoke-WebRequest http://localhost:8080/api/v1/auth/health

# 2. Check frontend API configuration
# File: InventoryClient/src/main/java/com/wafry/inventory/config/ApplicationConfig.java
# Should show: http://localhost:8080/api/v1

# 3. Rebuild frontend
cd "E:\Inventory Management System\InventoryClient"
mvn clean package
mvn javafx:run
```

### Database connection failed
```powershell
# 1. Check PostgreSQL service
Get-Service postgresql-x64-* | Start-Service

# 2. Test connection
psql -U postgres -h localhost -d wms

# 3. Create database if missing
psql -U postgres -h localhost -c "CREATE DATABASE wms;"

# 4. Verify database has tables
psql -U postgres -d wms -c "\dt"
```
