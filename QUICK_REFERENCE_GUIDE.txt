🔐 LOGIN FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Start Application
   ↓
2. Login Screen Appears
   ├─ Enter Username (admin or user)
   ├─ Enter Password (password or user)
   └─ Click Login Button (or press Enter)
   ↓
3. Validation (Frontend)
   ├─ Username not empty
   ├─ Username min 3 characters
   ├─ Password not empty
   └─ If invalid → Show error, stay on login
   ↓
4. Authentication (Backend)
   ├─ Send credentials to API
   ├─ Backend verifies against database
   ├─ Generate JWT token
   └─ Return success/failure
   ↓
5. Success Path
   ├─ Store authentication token
   ├─ Navigate to Inventory Screen
   └─ Load products from API
   ↓
6. Inventory Screen
   ├─ Display product table
   ├─ Show 5 demo products (if no API)
   └─ Ready for operations

12 ENTITY CLASSES MAPPED TO DATABASE TABLES:

Table: users
├─ Fields: id, username, password, email, role, created_at
└─ Records: admin, user (2 test users)

Table: products
├─ Fields: id, name, price, stock, description, created_at
└─ Records: 5 demo products (Laptop, Mouse, Keyboard, Monitor, USB Cable)

Table: warehouses
├─ Fields: id, name, location, capacity
└─ Records: Central storage facility details

Table: locations
├─ Fields: id, warehouse_id, code, zone
└─ Records: Physical storage locations within warehouse

Table: stocks
├─ Fields: id, product_id, warehouse_id, quantity
└─ Records: Current inventory levels by product & warehouse

Table: stock_movements
├─ Fields: id, product_id, quantity, type, timestamp
└─ Records: In/Out transaction history

Table: suppliers
├─ Fields: id, name, email, phone, address
└─ Records: Vendor information

Table: purchase_orders
├─ Fields: id, supplier_id, order_date, total_amount, status
└─ Records: Purchase transaction history

Table: sales_orders
├─ Fields: id, customer_id, order_date, total_amount, status
└─ Records: Sales transaction history

Table: order_items
├─ Fields: id, order_id, product_id, quantity, price
└─ Records: Line items in orders

Enum: UserRole
├─ Values: ADMIN, USER, MANAGER
└─ Used in: users table

Enum: MovementType
├─ Values: IN, OUT, ADJUSTMENT
└─ Used in: stock_movements table

Issue: Login fails
├─ Check username/password (admin/password or user/user)
├─ Ensure server is running on port 8080
└─ Check internet connection to localhost

Issue: Products don't load
├─ Check server status
├─ Verify database connection
├─ Check API endpoint configuration
└─ Falls back to demo data if API unavailable

Issue: Search not working
├─ Type product name (case-insensitive)
├─ Clear search field to reset
└─ Ensure products are loaded

Issue: Can't edit/delete product
├─ Select a product row first
├─ Ensure selected product has valid ID
└─ Check for API connection errors
